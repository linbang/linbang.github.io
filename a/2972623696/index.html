二叉搜索树相关

发表于 2020-02-07 更新于 2020-02-09 分类于 算法题 ， 二叉树 本文已被戳过 次了 评论数：
本文字数： 7.2k 阅读时长 ≈ 7 分钟

二叉搜索树的几个特性：

  • 中序遍历序列一定是有序的（判断是否是二叉搜索树、recover binary search tree，第k小的节点）
  • 根节点的值一定大于所有左子树的值，小于所有右子树的值（找二叉搜索树最低公共祖先）

二叉搜索树同样可以用DFS、BFS进行遍历，一般考虑序列大小问题的时候，使用DFS比较容易一点。

3、Unique Binary Search Trees

题目

Given n, how many structurally unique BST’s (binary search trees) that store values 1 … n?

Example:

思路

这个题目要求给定一个数字n，判断1~n这n个数字可以组成多少个BST。

首先我们对根节点进行遍历，其实n个数字能组成多少个就是以1~n这n个数字为根节点，能构成多少个BST。

所以考虑使用动态规划。

  • 我们以i为根节点，那么左子树就是[1-i-1]，右子树就是[i+1，n]。对于左子树存储[1…i-1]，假设可能的个数是f[i-1]，右子树存储的个数实际上与[1,n-i]相等，所以dp[i]=dp[i-1]*dp[n-i]
  • dp[i]：表示1…n存储的BST的数量
  • 初始化：dp[0] = 1; dp[1] = 1
  • 状态转移方程：dp[j] += dp[i-1]+dp[j-i]( for j in 2…n for i in 1…j)
  • return：dp[n]

题目

Given an integer n, generate all structurally unique BST’s (binary search trees) that store values 1 … n.

Example:

思路

这个题目要求记录所有的BST的路径。

  1. 每一次都在一个范围内随机选取一个结点作为根。
  2. 每选取一个结点作为根，就把树切分成左右两个子树，直至该结点左右子树为空。

大致思路如上，可以看出这也是一个可以划分成子问题求解的题目，所以考点是动态规划。
但具体对于本题来说，采取的是自底向上的求解过程。

  1. 选出根结点后应该先分别求解该根的左右子树集合，也就是根的左子树有若干种，它们组成左子树集合，根的右子树有若干种，它们组成右子树集合。
  2. 然后将左右子树相互配对，每一个左子树都与所有右子树匹配，每一个右子树都与所有的左子树匹配。然后将两个子树插在根结点上。
  3. 最后，把根结点放入链表中。

5、二叉搜索树的第k个节点

题目

给定一棵二叉搜索树，按节点值从小到大排列，找出其中的第k（k≥1）个结点。

思路

首先二叉搜索树，一定要联想到中序遍历序列是有序的。

因此我们找第k个节点，就是找中序遍历序列的第k小的数字。完全可以在中序遍历中找到。

时间复杂度O(N)

变种

如果题目变成求二叉搜索树的第k大的数字呢？

比较有意思，因为我们中序从左到右遍历只能找最小的第k个数。因此如果找最大的第k个数，应该从又到左遍历，以就是先遍历right，然后再遍历left。
