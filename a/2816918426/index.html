容器需要掌握的知识

发表于 2019-03-12 更新于 2020-02-09 分类于 JAVA ， JAVA容器 本文已被戳过 次了 评论数：
本文字数： 1.4k 阅读时长 ≈ 1 分钟

容器主要包括 Collection 和 Map 两种，Collection 和 Map 都是接口，Collection 存储着对象的集合，而 Map 存储着键值对（两个对象）的映射表。

Set

  • TreeSet：基于红黑树实现，支持有序性操作，例如根据一个范围查找元素的操作。但是查找效率不如 HashSet，HashSet 查找的时间复杂度为 O(1)，TreeSet 则为 O(logN)。
  • HashSet：基于哈希表实现，支持快速查找，但不支持有序性操作。并且失去了元素的插入顺序信息，也就是说使用 Iterator 遍历 HashSet 得到的结果是不确定的。
  • LinkedHashSet：具有 HashSet 的查找效率，且内部使用双向链表维护元素的插入顺序。

List

  • ArrayList：基于动态数组实现，支持随机访问。数组默认大小是10，如果超过需要调用grow()方法扩容，变成旧容量的1.5倍。
  • Vector：和 ArrayList 类似，但它是线程安全的。
  • LinkedList：基于双向链表实现，只能顺序访问，但是可以快速地在链表中间插入和删除元素。不仅如此，LinkedList 还可以用作栈、队列和双向队列。

ArrayList与LinkedList比较

1、ArrayList基于动态数组实现，LinkedList基于双向链表实现

2、ArrayList支持随机访问，LinkedList只支持顺序访问

3、LindedList在任意位置添加删除元素更快

LinkedList插入时间复杂度低，读取时间复杂度高。

ArrayList相反。

Queue

  • LinkedList：可以用它来实现双向队列。
  • PriorityQueue：基于堆结构实现，可以用它来实现优先队列。

4.HashMap、HashTable、ConcurrentHashMap

HashMap存储的是key-value键值对的形式，这里面有一些需要掌握的知识点。

一个Entry数组，每个数组就可以认为是一个桶，用来存放key的hash值。

1、头插法

对于每个hash值（桶），后面跟着的都是一个链表。

在put一个键值对的时候，首先计算key的hash值，然后对16（map大小）取模，如果已经有了相同的hash值了，就把这个新的值插到头节点上。

2、扩容

因为我们希望map达到O(N)的时间复杂度，如果链表很长，那么时间复杂度就比较高。所以这个时候就需要扩容。扩容时需要重新计算桶下标。这里计算桶下标的快捷方法，如果原来下标是0，还是0，否则+16。

初始可以放16个元素，如果达到上限，X2扩容。

3、链表变成红黑树

当链表长度>8的时候，自动变成红黑树的结构。

红黑树是一种自平衡的二叉查找树。用来解决链表长度过大的问题。快速的实现访问。

4、和CurrentHashMap区别

CurrentHashMap用了分段锁segment，每个分段锁维护几个桶，实现了线程安全，并发度更高。

比如在执行put操作的时候，如果发现hash的桶没有被上锁，就获取到这个segment锁；如果发现被锁上了，才会等待。

默认情况segment由16个，也就是说每个hash结果都有一个segment锁。

5、与HashTable的区别

  • HashMap和HashTable都是map结构，只不过HashTable是synchronization（同步）的，线程安全的。
  • HashMap不能保证map中元素次序，hashtable可以保证次序。
