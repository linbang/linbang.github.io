最大子数组乘积

题目

Given an integer array nums, find the contiguous subarray within an array (containing at least one number) which has the largest product.

Example 2:

思路

最大子数组乘积问题，首先考虑使用dp。但是这里有个问题就是可能有负数出现，负负得正，值又会变大了。因此肯定不能值用一个数组来保存结果了。

我们考虑使用两个dp数组，dp1[i]表示以 i 结尾的连续数组乘积最大，dp2[i]表示以 i 结尾的连续数组乘积最小。

所以我们在遍历的时候

1、dp1[i] = max{dp1[i-1]*nums[i], dp2[i-1] * nums[i], nums[i]}

2、dp2[i] = min{dp1[i-1]*nums[i], dp2[i-1] * nums[i], nums[i]}

3、max = max{max, dp1[i]}

因为我们不需用保存之前的状态，所以可以用两个变量来缩减使用空间。

二维数组最大子矩阵和

问题

给定一个矩阵Matrix, 其中的值由正, 有负, 有0, 返回子矩阵的最大累加和.

例如: 给定矩阵
| -90 | 48 |78 |
| 64 | -40 | 64 |
| -81 | -7 | 66 |

累加和最大的子矩阵为
| 48 | 78 |
| -40| 64 |
| -7 | 66 |

思路

不同于数组, 矩阵需要在两个维度上进行移动, 所以逐个子矩阵尝试的方法, 复杂度很高.
一个思路是, 将矩阵的最大累加和转换为数组的最大累加和.
以矩阵
| -2 | 3 | -5 | 7 |
| 1 | 4 | -1 | -3 |
为例
1 第一行作为一个数组, 求它的最大子数组累加和 7
2 第二行作为一个数组, 求它的最大子数组累加和 5
3 将矩阵的两行数字对应相加, 得到数组 | -1 | 7 | -6 | 4 |, 求这个数组的最大累加和为7
4 求所得累加和中最大的值, 即为7.
也就是说, 在矩阵中, 限定某k行, 在必须含有k行元素的情况下, 可以通过将每一列的k个元素累加生成一个累加数组, 然后求出这个数组的最大累加和max, 就是必须含有k行元素的子矩阵中的最大累加和.

题目

Given an array of integers and an integer k, you need to find the total number of continuous subarrays whose sum equals to k.

Example 1:

思路

题目中数组无序，要求找到所有的子数组和为k的个数。

思路一：两层循环

第一个想法就是两层循环暴力搜，时间复杂度比较高。

思路二：map辅助

Map保存<前缀和，出现的次数>

注意这里map的记录过程必须放在循环中，因为本质上来说，我们是对每个位置，去找它左边是否有sum-k这个子数组，如果事先保存，那么很可能记录了右边的情况。

上面题目中数组是无序的，如果数组有序，要求找到所有和为K的连续子数组，那么就要使用双指针+滑动窗口法来求解了。

正整数数组中最短的连续子数组和为K

Minimum Size Subarray Sum 这里每个数字都是正整数

第一种解法：

滑动窗口法，O(n)

第二种解法：

建立sum直方图，然后for循环过程中，进行二分查找，O(logn)

比如数组：[3,2,1,4,2,5]，和数组就是：[0,3,5,6,10,12,17]。

直观上看，我们发现10-3=7，17-10=7，所以对于3，我们就去找右边是3+7=10的数字，对于10，我们就去找右边10+7的数字。

因此我们可以得出结论，首先一遍for循环每个数字，然后在这个nums[i]的右边数组中去找nums[i]+target的数字。如果有这个数字，就说明i->j和是target，就可以得到结果了。

这里需要保证和数组是递增，所以原数组中每个数字都得是正整数。

这个题目能用双指针和二分，本质是因为数组是正整数。如果去掉这个限制，就不能用这些方法了。

如果题目变成求正整数数组中和为target的最小子数组长度，我们可以建立和数组，然后用2-sum的思想，将数字与下标对应，一次遍历，就可以找到targe+sum[i]是否存在了。

数组能否切分成两个和相同的子集合

Partition Equal Subset Sum

1、首先计算数组和sum，我们要找的target就是sum/2，下面去找是否有子序列和为target。

（数组中找几个数字和为target！！典型的DFS啊。当然也可以用coin change2中的dp思想来做）

2、考虑到子序列，想到使用DP。dp[i]表示和为i的子集合是否存在，最后我们只需要return dp[target]

3、dp[i] = dp[i] || dp(i-x) ( if i-x >= 0 for x in nums)

数组能否切分成K个和相同的子集合

Partition to K Equal Sum Subsets.

将数组分割成k个子序列，每个子序列的和相同。不能用DP来做，因为验证了一个，我们还得去验证其他的。

考虑使用DFS，同时用visited数组保存已经访问过的数字。

数组切分成m个子数组，求和最大的连续子数组最小

Split Array Largest Sum

二分查找是神仙玩的。。。好好看懂DP算法把。

1、$dp[i][j]$表示前 j 个元素要分成 i 组，所能得到的最小的各个子数组中的最大值。初始化为Integer.MAX_VALUE。

2、建立累加和数组

3、如果前 j 个元素要分成 i 组，最多情况是j组，最小情况是1组，所以i in range(1,j)；对于任意一个情况k，$dp[i-1][k]$代表将数组中前k个元素分成 i - 1 组所能得到的最小的各个子数组中的最大值。

我们可以举个例子考虑一下，

首先我们计算得到$dp[1][1]$表示前1个元素被分成1组所能得到的最大和最小，下面我们去看$dp[1][2]$表示前2个元素被分到一个组里面最大和最小。。。所以我们的边界条件$dp[1][j]=sum(0,j)$。

对于$dp[i][j]$，我们用一个变量k从 0~j 进行遍历，我们要去找的就是找$max{dp[i-1][k]} , sum(k+1…j)$中的最小值。因此我们可以得到状态转移方程：

$dp[i][j] = min{max(dp[i-1][k], sum(k+1,…, j)} 0<k<=j;$
