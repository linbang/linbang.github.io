面向对象：继承、多态、封装

Java中继承的理解

继承就是字类继承父类的实例和方法，字类通过extend父类，就可以实现对父类的继承。

注意父类中private方法和实例是无法被继承的。

继承的子类可以重写父类的方法。

Java中多态的理解

含义

多态，程序中定义的引用变量指向哪个类的对象在编程时不确定，只有在程序运行时才确定。因为运行时，变量才确定到具体的类上面。

必要条件

1、继承：必须存在有继承关系的子类和父类

2、重写：字类必须重写父类的方法，这样在运行时才会调用字类的方法

3、向上转型：必须将子类的引用赋给父类对象，只有这样该引用才能够具备调用父类的方法和子类的方法。

实现

实现1，基于继承的实现

比如我们有peple父类，父类中有一个public的函数叫getname。

有一个man子类和woman子类，man子类重写了getname方法并输出man，woman类重写了getname方法并且输出woman。

main函数中，我们令peple peple1 = new man();peple peple2 = new woman();然后分别调用getname的方法，分别会输出man和woman，这个过程就叫做基于继承的多态实现。

实现2，基于接口的实现

因为一个子类只能继承一个父类，有限。而通过接口就可以将不同的服务进行封装，一个类可以实现多个接口。

声明一个接口用interface关键字，接口不能实例化，不能 new 接口。抽象类可以new一个对象。

接口的实现用implements关键子，抽象类的继承用extend。

int和Integer的区别

int是Java中的原始数据类型，Java为每个原始类型都提供了封装类，Integer就是java为int提供的封装类，所以Integer需要new一个对象。

但是int a = new Integer(100);这种语法是对的，因为Java中会对这种情况自动进行拆箱和装箱。

‘==’和’.equals()’的区别

‘==’的理解

1、比较基本数据类型的时候，比如int、double、char等，用==比较的是他们的值是否相等

2、比较引用数据类型的时候，比较的是他们在内存中存放的地址

所以比较两个对象，如果对象值都一样，但是是分别new的，那么用==肯定是false，因为每new一次，就会新开辟一块堆内存空间。

‘.equals()’的理解

Java所有的类都继承Object类，.equals()就是object类中的方法。

如果类不对.equals()方法进行重写，那么它比较的也是堆中的地址，作用和’==’相同。

但是一般情况下，我们在类中都会重写.equals()方法，比如string是一个类对象，它就重写了.equals方法用来比较两个字符串的值是否相等，所以我们比较两个字符串的时候一定要用s1.equals(s2)。

同时在自己定义类，并且可能需要比较类对象是否相同的时候，也最好重写这个方法。

堆区、栈区、方法区等等

JVM的内存可以分为堆区、栈区和方法区。简单总结一下可能会问的东西。

堆区

堆区全部存放对象，jvm只有一个堆区，可以被所有线程共享。

堆区是gc回收的一个重要区域

栈区

每个线程都有一个栈区，用来保存基本的数据类型、对象的引用。

每个栈中的数据都是线程私有的，其他线程无法访问。

栈分为三个区：基本类型变量区、执行环境上下文、操作指令区

方法区

也叫静态区，所有的线程共享，包含所有的class和static变量

方法区中包含的都是在程序中永远唯一的元素

JDK1.8中，已经没有方法区了，将方法区直接放在一个与堆不相连的本地内存区域，叫做元空间。

Java设计模式

单例模式

因程序需要，有时我们只需要某个类同时保留一个对象，不希望有更多对象，此时，我们则应考虑单例模式的设计。

有很多种实现：

1、懒汉模式

来一个线程就获取一个对象，导致实例化多次类对象，不符合单例模式的要求。

2、懒汉式（线程安全）

对get方法加了一个synchronized锁，当一个线程进入该方法是，其他的线程必须等待，这样会引起阻塞。

3、双重校验锁（线程安全）

uniqueInstance 只需要被实例化一次，之后就可以直接使用了。加锁操作只需要对实例化那部分的代码进行，只有当 uniqueInstance 没有被实例化时，才需要进行加锁。

双重校验锁先判断 uniqueInstance 是否已经被实例化，如果没有被实例化，那么才对实例化语句进行加锁。

这里要判断两次if(singleton==null)

1、第一次校验，如果已经有这个对象了，就直接返回，没有的话，进入语句块。

2、如果没有第二次判断，那么两个线程同时进入下面语句

第一个线程进来后生成一个对象，第二个线程进来后又生成一个对象。所以就不对了。

niqueInstance 采用 volatile 关键字修饰也是很有必要的， uniqueInstance = new Singleton(); 这段代码其实是分为三步执行：

  1. 为 uniqueInstance 分配内存空间
  2. 初始化 uniqueInstance
  3. 将 uniqueInstance 指向分配的内存地址

但是由于 JVM 具有指令重排的特性，执行顺序有可能变成 1>3>2。指令重排在单线程环境下不会出现问题，但是在多线程环境下会导致一个线程获得还没有初始化的实例。例如，线程 T1 执行了 1 和 3，此时 T2 调用 getUniqueInstance() 后发现 uniqueInstance 不为空，因此返回 uniqueInstance，但此时 uniqueInstance 还未被初始化。

使用 volatile 可以禁止 JVM 的指令重排，保证在多线程环境下也能正常运行。

4、静态内部类实现

当 Singleton 类加载时，静态内部类 SingletonHolder 没有被加载进内存。只有当调用 getUniqueInstance()方法从而触发 SingletonHolder.INSTANCE 时 SingletonHolder 才会被加载，此时初始化 INSTANCE 实例，并且 JVM 能确保 INSTANCE 只被实例化一次。

这种方式不仅具有延迟初始化的好处，而且由 JVM 提供了对线程安全的支持。

Java中锁的理解

Java中面对高并发，多个线程处理一块资源的时候，为了维护数据的一致性，会使用synchronized关键字来给资源上锁。只有拿到锁的线程才有权限访问资源，加锁保证了资源访问的顺序性。

乐观锁：

认为竞争不总是发生，不许用加锁。如果发生冲突，就会进入到一个while循环中，不停的尝试获取资源。

（这里不挂起，因为阻塞之后就会将程序从用户空间转到内核空间，非常浪费资源）

悲观锁：

认为竞争总是会发生，所以进程都某个资源进行操作时，会加synchronized关键字进行上锁，防止其他的进程获取这个资源。

overload和override的区别

overload是重载的意思，override是覆盖，也就是重写的意思。

重载就是指一个函数名，可以设置不同的参数来实现重载。（注意，只是改变返回类型不算重载，无法通过编译）

重写，是指子类的方法可以对继承的父类的方法进行重写。比如我们经常会在集合中重写compare方法，例如大顶堆等，这些就叫做override。

缓存池

new Integer(123)和Integer.valueof(123)的区别：

  • new Integer(123)每一次都会新建一个对象
  • Integer.valueof(123)会使用缓存池中的对象，多次调用取得同一个对象的引用

抽象类和接口

抽象类

抽象类和抽象方法都通过abstract关键字进行声明，抽象类中包含抽象方法，抽象方法一定定义在抽象类中，而且一般情况抽象方法不会写具体的实现。

抽象类不可以被实例化。

接口

接口是更抽象的抽象类，用interface声明。接口中的方法都不能写实现。

接口中成员默认都是public，字段默认是static和final。

super关键字

用于子类访问父类的构造函数，或者通过super访问父类的成员变量。

static关键字

static可以用来修饰很多东西，比如静态变量、静态方法、静态内部类等等

静态变量

  • 又称为类变量，也就是说这个变量属于类的，类所有的实例都共享静态变量，可以直接通过类名来访问它。静态变量在内存中只存在一份。

静态方法

  • 静态方法不依赖于类对象就可以进行访问，对于静态方法是没有this指针的，因为他不依赖于对象。
  • 静态方法中不能访问类的非静态成员变量和非静态方法。

静态内部类

  • 静态内部类可以在类方法中直接new一个对象，而非静态内部类依赖于外部类的实例。

final关键字

final声明的常量，不能被修改。

final声明的方法，不能被子类重写。

final声明的类，不能被继承

Java反射机制

每个类都有一个 Class 对象，包含了与类有关的信息。当编译一个新类时，会产生一个同名的 .class 文件（字节码文件），该文件内容保存着 Class 对象。

所谓的反射机制，是指代码在JVM中运行的时候，jvm获取到类中所有的属性和方法，对于一个对象，都能够调用它的任意一个方法。

JKD和JRE

jdk是指java程序用的开发包，比如各种方法、泛型的实现。

jre是指java运行的环境，比如jvm。

String类是否可以被继承

不可以被继承

String类使用final修饰，final修饰的类无法被继承

final类不能被继承；

final方法不能被重写；

final变量不能被修改；

synchronized和volatile

两个关键字都是为了解决java高并发的问题。

synchronized修饰方法或者代码块

volatile修饰变量，线程在每次使用变量的时候，都会去读取变量修改后的值。

volatile轻量级同步，不会加锁导致线程阻塞，如果变量是volatile修饰，每一次读和写，都必须和主内存进行交互，同时这个关键字还禁止了编译器和处理器的一些重排序优化。（所以在双重锁实现单例模式的时候需要加volatile，防止jvm的虫重排序）

volatile最常使用就是单例模式的双重锁实现，有时也会对boolean变量进行加volatile。

堆、栈溢出

JVM中堆保存着所有的对象，堆溢出就是不断的new对象，就会溢出。

JVM中栈是线程私有的，保存一些引用、变量、常量等，每执行一次方法就会加一个栈帧（栈顶元素）。栈溢出可以理解为方法执行时创建的栈帧超过了栈的深度，最有可能的就是递归。

泛型

Java中泛型广泛应用在集合类框架中，泛型可以简化代码。集合定义中只要通过<>定义类型就行了。

泛型通过类型擦除实现，在程序编译期间擦出所有类型，只保留Object累。判断类型信息是在写代码中，编译器在编译之间就会检查泛型是否正确，然后才可以编译。

异常

异常（Exception）是程序在执行过程中的错误，比如输入错误、数组边界错误、空指针错误等。

如果有异常捕捉代码，比如（try-catch），就会进入异常程序，不会影响程序运行；如果没有，或者throw异常，就可能会导致程序中断。

exception：可预见的错误，可以通过捕捉来控制。

error：不可预见的错误，比如堆栈溢出等，不可捕捉，jvm层面的错误，不可捕捉。
