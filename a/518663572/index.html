前言

前面章节介绍了requests库的使用，可以通过requests的get或者post方法获取url信息，实现了最简单的爬虫。但是获取到的只是string类型，还需要将其变成网页的各种元素，方便我们查找。这个变化的过程就用到了BeautifulSoup这个库，可以提取HTML文件中的数据。

解析器 使用方法 优势 劣势
Python标准库 BeautifulSoup(markup, “html.parser”) Python的内置标准库、执行速度适中 、文档容错能力强 Python 2.7.3 or 3.2.2)前的版本中文容错能力差

一般情况下，使用第一个和第二个解析器就足够了。

概念

网页元素都是由标签组成的，内容。我们获取的时候，可以直接使用.name,.attrs,.string等来获取想要的元素内容。

BeautifulSoup是将网页所有元素解析成树状，每个节点都是一个python对象。

一个简单的HTML是树形的，有三种遍历方式。比如：

1、下行遍历

属性 说明
.contents 子节点的列表，将所有儿子节点存入列表
.children 子节点的迭代类型，与.contents类似，用于循环遍历儿子节点
.descendants 子孙节点的迭代类型，包含所有子孙节点，用于循环遍历

2、上行遍历

属性 说明
.parent 节点的父亲标签
.parents 节点先辈标签的迭代类型，用于循环遍历

3、平行遍历

属性 说明
.next_sibling 返回按照HTML树顺序的下一个平行节点标签
.previous_sibling 返回按照HTML树顺序的上一个平行节点标签
.next_siblings 迭代类型，返回按照HTML树顺序的后续所有平行节点标签
.previous_siblings 迭代类型，返回按照HTML树顺序的前序所有平行节点标签
  • 平行节点的概念是指具有同一个父亲节点
  • 所有的遍历方式都可能出现string类型，要在代码中进行排除

方法

在解析完成后，我们就会获得一个soup对象，这个对象有很多方法来获取网页信息。

find函数

find函数是用来寻找标签用的，包含find（查找第一个标签），find_all（查找所有标签）等方法。

find_all( name , attrs , recursive , string , **kwargs )

  • name参数：要查询的标签
  • attrs参数： 定义一个字典参数来搜索包含特殊属性的标签
  • recursive参数：定义是否允许查询当前标签的所有子孙节点，默认是true
  • limit参数：定义找到若干个符合条件的标签之后就返回，不再继续寻找
  • string参数：直接搜索string字符串，接受字符串、正则表达式、列表、bool等

1、按name搜索：可以查找到所有名字为name的标签，忽略字符串对象。

2、按id搜索：如果标签中由id参数，那么可以直接搜索id来找到这个标签

id和class的区别

id是唯一的，class不唯一

3、按attr搜索：使用attrs参数定义的字典来搜索具有特殊属性值的标签

4、按css搜索：按照css类名搜索标签，但是class在python里是保留字，直接使用会导致语法错误，因此从BeautifulSoup 4.1.1版本开始，就使用class_参数搜索指定css类名的标签。

5、按string搜索：传入string参数可以直接搜索文档中字符串内容。

一般情况下，通过搜索“标签+属性”我们就可以找到想要的元素了。
