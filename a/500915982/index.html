单链表翻转题目总结

1、Reverse Nodes in k-Group

题目

Given a linked list, reverse the nodes of a linked list k at a time and return its modified list.

k is a positive integer and is less than or equal to the length of the linked list. If the number of nodes is not a multiple of kthen left-out nodes in the end should remain as it is.

Example:

Given this linked list: 1->2->3->4->5

For k = 2, you should return: 2->1->4->3->5

For k = 3, you should return: 3->2->1->4->5

Note:

  • Only constant extra memory is allowed.
  • You may not alter the values in the list’s nodes, only nodes itself may be changed.

思路

思路一：直接用多个指针翻转链表，时间复杂度O(N)，空间复杂度O(1)

这个题应该是链表翻转类型题目中最难的一个了，要求将连续k个节点进行翻转。这种翻转链表类型的题目，首先就考虑加上一个dummy节点，这样就更一般化了。然后考虑翻转中需要用到的几个边界节点，并且给他们进行标号。

下面我们来总结一下这个题的思路：

1、首先新建一个dummy节点，作为新的头节点。

2、进入一个while(cur!=null)的循环，这个循环里进行计数，如果到了k次，那么就调用reverse函数将head开头的k个节点进行翻转，然后重新计数。

3、对于reverse函数的设计，一方面参考传统的链表反转的思路，另一方面又要考虑到题目的要求。在翻转的时候，我们核心的其实要找到node边界的情况，比如0->1->2->3->4这种，我们需要最终让0->3，1->4，这才是题目要的关键，其他的就是链表内部翻转的代码了。

talking is cheap。。

思路二：用栈来协助翻转，时间复杂度O(N)，空间复杂度O(N)

因为链表翻转很容易用栈来实现，所以我们用栈来处理链表翻转。

栈从头节点开始不停的存元素，并且当栈大小是k个时候将这k个元素输出，自然的就形成了翻转。这里要注意如果最后栈中间剩下的元素<k个，那么这些元素要顺序输出。再将这些元素翻转就行了。

思路

思路一：栈，时间复杂度O(n)，空间复杂度O(n)

利用栈的先进后出的思想，将每个链表顺序保存在栈中，然后再一次pop出来。

思路二：递归，时间复杂度O(n)，空间复杂度O(1)

利用递归的思想。

注意如果递归很深会引起栈溢出。

思路三：双指针，时间复杂度O(n)，空间复杂度O(1)

这个方法最好，pre指针指向前一个位置，p指针指向当前位置，next指针指向下一个位置。

思路也比较好想，链表翻转类型的题目，首先就想到用多个指针来做。
